Building the Binary

# Compile to bin/api with version string injected
make build

# Equivalent command:
go build -ldflags "-X main.Version=$(git describe --tags --always)" -o bin/api .

# Run the binary directly
./bin/api

The Version variable in main.go gets the git tag at build time. It appears in the /health and / responses so you know exactly which version is running.

Dockerfile

# Multi-stage build: builder stage compiles, final stage is minimal
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY go.mod go.sum ./
RUN go mod download          # cache dependencies as a layer
COPY . .
RUN go build -o bin/api .

FROM alpine:latest           # minimal final image (~10MB)
COPY --from=builder /app/bin/api /api
ENTRYPOINT ["/api"]

Multi-stage builds keep the final image small — no Go compiler, no source code. Only the binary.

GitLab CI Pipeline

Defined in .gitlab-ci.yml. Key stages:

stages:
  - test      # go test ./...
  - build     # docker build + push to registry
  - deploy    # update Kubernetes deployment

# Deploy stage uses environment-specific variables
deploy:
  script:
    - kubectl set image deployment/lp-coupon-api
        api=$CI_REGISTRY_IMAGE:$CI_COMMIT_SHA

The image tag is the commit SHA — every push creates a unique, traceable image.

Blue/Green Deployment

Blue/green keeps two identical environments. Traffic switches between them at deploy time, enabling instant rollback.

Blue environment  (currently live — receiving traffic)
Green environment (new version deployed here)

Deploy steps:
1. Deploy new version to Green (blue still serves traffic)
2. Run smoke tests against Green
3. Switch load balancer: 100% traffic → Green
4. Green is now live, Blue is idle (rollback target)

Rollback:
- Switch load balancer back to Blue (seconds, not minutes)
- No re-deploy needed
Zero-downtime: Traffic switch is instantaneous. No requests are dropped. In-flight requests on Blue complete normally before Blue goes idle.

In Kubernetes, this is implemented with two Deployment objects and a Service selector that points to one label (slot: blue or slot: green). Switching means patching the Service selector.

Key Takeaways

  • make build injects git version into the binary via ldflags
  • Multi-stage Dockerfile: compile in Go image, run in Alpine (~10MB final image)
  • CI pipeline: test → build Docker image (tagged with commit SHA) → deploy
  • Blue/green: two environments, switch traffic instantly, rollback = switch back
